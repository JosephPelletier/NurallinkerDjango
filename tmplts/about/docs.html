Browsing

The Browse section lets you see all the linkers made by everyone on the platform. On each linker you will see its name, its author's name, and its description. By clicking a button on the liner you can start it or download it so you can modify and re-upload it as your own, but please do credit the uploader in the description of your upload. You may also click the name of the uploader of the linker to see everything they've uploaded.

Using a linker

Click the name of the linker or click the 'Start' button to start practicing.

You will be shown a question in the form of text image or sound, you will have to click the 'Play' button to hear the sound. If you have disabled javascript you will have to use your browser's built in audio controls, if you don't know that that means then you can safely ignore it.

You will also be given a choice of a few answers, there can be 2 to 9 choices. The answer choices can be text, image, or sound; or the answer can be open ended, in this case you will have to recall the text answer from memory and enter it. To select an answer just click it, clicking the 'Play' button on an audio answer will not select it, but clicking anywhere else on an audio answer will select it. When an answer is selected the circle belonging to it will be filled. Once you are satisfied with your answer click the 'Check' button to check your answer, if you click the 'Check' button without selecting or entering an answer the question will be skipped and counted as wrong.

Then you click the 'Check' button you will be show the results page. You will see the following things in the following order.

  1. The question you were asked
  2. The result; Correct, Incorrect, or Skipped
  3. The correct answer for the question you were asked
You will not be shown the answer you choose if it was wrong, that won't help you remember the right answer. From this page click next to be shown another question.

There is not a set duration for a linker, the number of questions you are asked is a combination of how well you are doing, how many questions the linker has, and the randomness of what question you are asked. See Question generation and Mastery for more on this.

You may user more than one linker at once and even close them without losing your progress, but there is no guarantee your progress will be saved day to day (this may change in the future, probably when we add a progress measure to the check page)

Question generation

Questions are generated with the help of sudo-random numbers, firstly a bit is generated, based on this bit the question generator either selects the question or the correct answer. Each possible question, and each possible answer has an integer weight called its 'odds'. The odds control how likely a particular question or answer is to be selected, if question 'A' has odds 1 and question 'B' has odds 3, question 'B' is three times more likely to be selected than question 'A'. The same is true if it were answer 'A' and 'B' instead of question 'A' and 'B'

When a question is answered correctly the odds number of the asked question and the correct answer are decreased by one, likewise when a question is answered incorrectly or is skipped the odds number of the asked question and the correct answer are increased by one; this increases the chance that it will be seen again as a question, and the chance it will again be the correct answer. The odds number starts at 2 and is not aloud to fall bellow 1. When the odds humber on every question and every answer is 1 the the linker is mastered. See Mastery for more info on mastery.

Mastery

When it seems that you are reliably getting every question right you will see a page telling you that you have mastered the linker, at this point you probably don't need to study any more for now, but you may continue by clicking the name of the linker or by clicking the word 'here'. When you see this page you progress on the linker will be reset so that you can use it again without being shown the mastery page immediately.

For technical information on when exactly mastery occurs see Question generation

Loging-in

To get to the login page click 'Manage' on the nav-bar while logged-out. To log-in or create an account enter a username and password then click the 'Login' button. Usernames may not begin with a dot or contain a forward slash. If the account name is available to be created you will be prompted to confirm the password for the new account and if the passwords match you will be logged-into the newly created account.

The login page is made to be error tolerant, if the user types an invalid username or incorrect password an error message will appear above the problematic field.

Passwords are not stored, what is stored is the sha256 digest of the concatenation of the username with a forward slash with the password `sha256(uname .. '/' .. passwd)`. Since the forward slash is not aloud in usernames this authentication string is guaranteed to me unique and thus it is astronomically unlikely that two authentication hashes are the same. This makes these passwords useless to an attacker who has stolen them from the server to the extent to which they are too strong to be brute forced. Since all auth strings are unique no matching password attack can be preformed to identify passwords. The system maintains that brute force is the only possible attack. Many site designers skip over good security and store your passwords in one form or another in order to authenticate you. When you log-in to this site your auth-string is produced, hashed, and immediately discarded, if the hash matches the one stored on the server, then you, beyond any reasonable doubt know your password, and so, you are aloud to log-in.

Linker Settings

The linker settings allow you to customise your linkers description, what types of questions and answers are enabled or disabled, and how many multiple choice options will be given for a given answer type.

nurallinker will ignore lines starting with '#', these lines are called comments. if you add any lines they have to start with '#' the rows that control the settings do not start with '#'

the settings are organized as a grid, the rows are q (question), a (answer), c (choose) and the columns are i (image), a (audio), t (text), o (open)

    for settings that are switches + is enabled, - is disabled
  • row q is for question switches, it controls what question types are enabled, + is enabled, - is disabled, there are 3 question switches: i, a, t
  • row a is for answer switches, it controls what question types are enabled, there are 4 answer switches: i, a, t, o
  • row c is for choose numbers, instead of a + or - for on or off you place a single digit from 2 to 9, that will be the number of choices given for the corresponding answer type. If the type is disabled place a 0 or - in place of the digit for readability
  • column i is for the image question/answer type
  • column a is for the audio question/answer type
  • column t is for the text question/answer type
  • column o is for the open-ended question/answer type, open ended is a text answer but the user types it, no choices are given.
  • row d holds the decription of the linker as it shows up on the browse page.
    • While the order of the rows is flexible, the order of the columns is not

# this is an example settings file
# i a t o
q + - +
a + + + -
c 3 0 3
d this is an example linker

Identifiers

Identifiers map questions to answers and vice versa. The part of the file name before the first dot is the identifier. For example, 'dog.sitting.png' would match with 'dog.txt' and 'dog.barking.mp3' because they all have the identifier 'dog'.

Linker Requirements

If a linker requirement is not satisfied, an update error occurs

  • text, image, and audio directories, as well as the settings.txt file, must me at the root of the zip
  • No extra file or directory may be at the root of the zip
  • You may only include files in the aloud formats for any directory in the linker
    Aloud formats:
    • text: .txt
    • image: .jpeg, .jpg, .png
    • audio: .mp3
  • Every enabled question must have a corresponding answer in each enabled answer type
  • Every enabled answer must have a corresponding question in each enabled question type
  • choose setting for any given answer type must be less or equal to the amount of answers identifiers for the answer type
  • the q setting should have exactly 3 switches: i, a, t; - or +
  • the a setting should have exactly 4 switches: i, a, t, o; - or +
  • the c setting should have exactly 3 digits: i, a, t; from 2 to 9
  • the q, a, c, and d setting must all be present

Update errors

Update errors occur when a linker requirement is not satisfied, and appears on the update page above the file choose button.

[text/image/audio/settings.txt] is not present, it should be at the root of the zip

If you get this error chances are when you created the zip file you selected the folder of the linker instead of selecting the files in the folder of the linker. Try again but this time just select text, image, audio, and settings.txt then create the zip.

{file} should not be in the zip

This error occurs when an extra file is present in the zip root. There should be three directories: 'text', 'image', and 'audio' and a 'settings.txt' file and nothing else.

[text/image/audio] should be a directory

This occurs if you've made a file called 'text', 'image', or 'audio'; these should be folders not files.

settings.txt should be a file

This occurs if you've made a folder called settings.txt, this should be a file not a folder.

[text/image/audio] format not allowed: .{format}

Aloud formats:

  • text: .txt
  • image: .jpeg, .jpg, .png
  • audio: .mp3

{identifier} is a question but is not a [text/image/audio] answer

Every enabled question must have a corresponding answer in each enabled answer type.

{id} is an answer but is not a [text/image/audio] question

Every enabled answer must have a corresponding question in each enabled question type.

choose setting for [texts/images/sounds] (#) is greater than the number of [text/image/sound] answers (#)

The choose setting for any given answer type must be less or equal to the amount of answers identifiers for the answer type.

q is too [short/long], should have 3 switches

The q setting should have exactly 3 switches, i, a, t

a is too [short/long], should have 4 switches

the a setting should have exactly 4 switches, i, a, t, o

c is too [short/long], should have 3 digits

the c setting should have exactly 3 digits, i, a, t

c should have 3 digits, no other non-space characters

There is something on this line that is not a digit [2-9].

c should have 3 digits from 2 to 9

The digits 0 and 1 is are aloud if the the answer type is enabled.

[q/a/c/d] ([question/answer/choose/description]) missing

The q, a, c, and d setting must all be present.
